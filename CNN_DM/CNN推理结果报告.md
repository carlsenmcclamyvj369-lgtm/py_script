# CNN Mosquito Noise 推理结果报告

## 1. 概述

使用 4 层 CNN 模型对 32 张测试图片进行蚊式噪声 (Mosquito Noise / Gibbs Ringing) 检测。模型将每张图划分为 8×8 block，对每个有效 block（排除图像边界 4 个 block 的 CNN 感受野不足区域）输出 0~1 的概率值，以 0.5 为阈值判定是否为蚊式噪声。

## 2. 模型信息

| 项目 | 值 |
|------|-----|
| 模型文件 | `mosquito_denoise_cnn.pth` (137.8 KB) |
| 网络结构 | 4 层 Conv2D + BatchNorm + Sigmoid |
| 层配置 | 16→32→64→16→1, kernel=3, padding=0 |
| 输入特征 | 16 维 (方差、边缘、振铃、二阶差分等) |
| 输入大小 | 9×9 邻域 patch (16通道) |
| 训练数据 | DM: 9,960 patches (806,760 行), Not DM: 10,412 patches (843,372 行) |
| 损失函数 | BCELoss |
| 优化器 | Adam, lr=1e-3 |
| 训练轮数 | 40 epochs |
| 设备 | CPU (推理时) |

> 注：训练时使用 CPU，推理时在 CPU 上运行。每张 1080p 图片平均耗时约 40~60 秒。

## 3. 推理流程

1. 读取 BMP 图像，转为 BT.709 Y 亮度通道
2. 按 8×8 grid 计算 49 维特征图
3. 从中选取与训练一致的 16 个特征，按固定除数归一化到 [0,1]
4. 对每个非边界 block，提取其 9×9 邻域特征 (81×16)，reshape 为 (16,9,9) 张量
5. CNN 推理输出概率，> 0.5 判定为 DM
6. 在原图上叠加红色标记 DM block，绘制灰色网格线

## 4. 16 个输入特征

| 序号 | 特征名 | 说明 | 归一化除数 |
|------|--------|------|-----------|
| 1 | mean_var | 3×3 窗口方差均值 | 1020 |
| 2 | low_var_count | 低方差像素计数 (<100) | 64 |
| 3 | high_var_count | 高方差像素计数 (>500) | 64 |
| 4 | edge_strength | 边缘强度 (水平/垂直梯度均值较大者) | 255 |
| 5 | edge_orientation_conf | 边缘方向置信度 | 1.0 |
| 6 | col_ringing_mean | 列方向振铃评分均值 | 1.0 |
| 7 | row_ringing_mean | 行方向振铃评分均值 | 1.0 |
| 8 | second_diff_max | 二阶差分能量 (行列较大者) | 510 |
| 9 | second_diff_min_max | 二阶差分行列比值 | 1.0 |
| 10 | profile_ringing_mean | 所有轮廓振铃评分均值 | 1.0 |
| 11 | ringing_mean_min | 行列振铃均值中较小者 | 1.0 |
| 12 | ringing_mean_min_max | 行列振铃均值比值 | 1.0 |
| 13 | row_ringing_max | 行方向振铃评分最大值 | 1.0 |
| 14 | row_diff_max | 行均值差最大值 | 255 |
| 15 | col_ringing_max | 列方向振铃评分最大值 | 1.0 |
| 16 | col_diff_max | 列均值差最大值 | 255 |

## 8. 有标注 vs 无标注对比分析

### 8.1 统计对比

| 指标 | 有标注 (13张) | 无标注 (19张) | 全体 (32张) |
|------|:-----------:|:-----------:|:---------:|
| DM率 均值 | **13.5%** | **11.9%** | 11.0% |
| DM率 中位数 | **7.9%** | **6.9%** | 6.1% |
| DM率 标准差 | **12.7%** | **13.3%** | 12.5% |
| DM率 范围 | 0.2% ~ 36.2% | 0.2% ~ 46.8% | 0.2% ~ 46.8% |
| 总 DM / 总 block | 35,637 / 319,056 (11.2%) | 24,080 / 679,852 (3.5%) | 59,717 / 998,908 (6.0%) |

**关键差异**:
- 两组均值接近 (13.5% vs 11.9%)，标准差也接近 (12.7% vs 13.3%)，分布形态相似
- 加权平均 DM 率有标注组 (11.2%) 明显高于无标注组 (3.5%)，因为有标注组集中在中低分辨率 (大量 720p/1080p)，而无标注组包含较多 4K 大图 (block 多但 DM 率低)
- 说明模型对低分辨率内容检测更敏感，对 4K 高分辨率内容偏保守

### 8.2 分辨率分布差异

| 分辨率 | 有标注 (张) | 有标注 DM率均值 | 无标注 (张) | 无标注 DM率均值 |
|--------|:---------:|:------------:|:---------:|:------------:|
| 856×480 | 0 | — | 1 | **46.8%** |
| 720×480 | 0 | — | 2 | **34.5%** |
| 720×576 | 2 | **33.1%** | 1 | 22.2% |
| 1280×720 | 0 | — | 1 | 9.2% |
| 1920×544 | 1 | 0.2% | 3 | 9.0% |
| 1920×1080 | 10 | 10.9% | 8 | 8.5% |
| 3840×2160 | 0 | — | 3 | 1.4% |

有标注组主要集中在 1920×1080 (10张)，无标注组覆盖从 480p 到 4K 的更广范围。**DM 率与分辨率呈明显负相关**。

### 8.3 Hisense 序列一致性

同一来源的 Hisense 图片，1 张有标注、3 张无标注：

| 图片 | 标注 | DM率 |
|------|:---:|:----:|
| hisense_mnr_mis_clarity#0002 | 有 | 0.2% |
| hisense_mnr_mis_clarity#0007 | 无 | 0.8% |
| hisense_mnr_mis_clarity#0012 | 无 | 0.3% |
| hisense_mnr_mis_clarity#0017 | 无 | 0.2% |

四张图检测率高度一致 (0.2%~0.8%)，说明模型对同类内容输出稳定，该序列确实极少蚊式噪声。

### 8.4 结论

| 发现 | 说明 |
|------|------|
| 两组分布相似 | 均值/中位数/标准差接近，模型行为一致 |
| DM率与分辨率负相关 | 低分辨率内容 DM 率普遍更高 |
| 有标注组加权平均更高 | 因其不含 4K 低 DM 率大图 |
| Hisense 跨组验证一致 | 模型对同源内容稳定 |
| 无极端异常值 | 最高 46.8%(5.2.24.MNR) 和 39.0%(002_VideoResolution_1) 虽高但不离谱 |
